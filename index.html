How It Works

7 Scoring Signals

1. Semantic Similarity

Sentence embeddings (all-MiniLM-L6-v2, ~50ms/resume) - catches synonyms

2. Skills Semantic + 3. Skill Overlap

Embedding + Jaccard (70% recall, 30% Jaccard) on extracted skills from 635+ skill taxonomy

4. TF-IDF Keywords

sublinear_tf=True - keyword repetition saturates (anti-gaming)

5. Title + 6. Experience + 7. Education

Structured signals: job title similarity, piecewise experience match, 5-level ordinal education

Why not just one signal?

TF-IDF Only Misses "ML" = "Machine Learning" Embeddings fix
Embeddings Only Misses "must have PMP" TF-IDF + skills fix
LLM-as-Judge Primacy bias, 100x slower LLM for explain only

Smart weight auto-selection: Semantic profile (text-heavy JDs) vs Structured profile (credential-heavy JDs). Weights learned via differential evolution on labeled data.

Limitations & Roadmap

Known Limitations

  • No fine-tuning - generic embeddings. ConFit v2: +17.5% nDCG with domain tuning
  • Small eval set - 64 pairs, single annotator. Need 50+ JDs x 50+ resumes, 3 annotators
  • Bias risk - embeddings encode training biases. Fairness monitoring + swap reranking built in
  • Format sensitivity - regex section detection. Production: LayoutLMv3

Production Roadmap

Fine-tune bi-encoder, skill taxonomy, ATS integration, GPU deploy

Vector DB (FAISS/Qdrant) for 100K+ resumes, recruiter feedback loop

Bias audit (NYC LL144 / EU AI Act), A/B testing, monitoring, multi-language
