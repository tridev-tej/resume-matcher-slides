Core Design Decision

Why Hybrid?

TF-IDF Only Misses synonyms ("ML" ≠ "Machine Learning") Embeddings fix this
Embeddings Only Misses exact requirements ("must have PMP") TF-IDF + skills fix this
LLM-as-Judge Primacy bias, 100x slower, non-deterministic LLM for explain only
Hybrid (Ours) 7 signals — each covers the others' blind spots Best of all

Ref: Iso et al., NAACL 2025 — primacy bias in LLM-based resume screening  |  ConFit v2, RecSys 2024 — domain-specific beats general LLMs

Scoring Engine

7 Scoring Signals

1. Semantic Similarity

Sentence embeddings, cosine distance — catches synonyms and paraphrases

2. Skills Semantic

Embedding comparison of extracted skill lists — "React" ≈ "React.js"

3. Skill Overlap

70% recall + 30% Jaccard blend — exact skill matching with coverage weighting

4. TF-IDF Keywords

Sublinear TF for anti-gaming — repeated keywords saturate quickly

5. Title Match

Job title similarity via embeddings — "SWE" ≈ "Software Engineer"

6. Experience Match

Piecewise: 70%+ → 0.6 floor, 100-150% → 1.0, >150% → overqualification decay

7. Education Match

5-level ordinal scale (none → PhD). No penalty for exceeding requirement — higher degree is never a flight risk.

Robustness

Anti-Gaming Defenses

  • Skill cap at 40 — 99th percentile of real resumes. Beyond this, extra skills are ignored
  • Sublinear TF-IDF — sublinear_tf=True means keyword repetition saturates quickly
  • Multi-signal defense — stuffed resume scores high on keywords but low on semantic, experience, education

Honest Assessment

Known Limitations

No fine-tuning

Generic embeddings. ConFit v2 showed +17.5% nDCG with domain fine-tuning.

Small eval set

64 pairs, single annotator. Production needs 50+ JDs × 50+ resumes, 3 annotators.

Bias risk

Embeddings encode training data biases. Fairness monitoring in place, full debiasing deferred.

Format sensitivity

Regex-based section detection with fallback. Production: LayoutLMv3 for ML-based layout parsing.

English only

Multilingual embedding swap (BGE-M3) would extend to 100+ languages.

Weight generalization

Semantic similarity inflates between unrelated management roles. Per-domain weights help.
